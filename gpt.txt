
fn get_name_from_purple_name(name: &PurpleName) -> Result<Option<String>, Box<dyn Error>> {
    let mut name_parts: Vec<String> = vec![];
    let ser = serde_json: :to_string(&name).unwrap();
    debug!("{:?}", ser);
    if let Some(literal) = &*name.literal {
        name_parts.push(literal.to_string());
    }

    if let Some(variables) = &*name.variables {
        for variable in variables {
            match &*variable.key {
                Some(key) if key == "adjective" || key == "1" => {
                    if let Some(var_value) = &*variable.value {
                        if let Some(value_key) = &*var_value.key {
                            let name = render_name(value_key.clone().trim().replace("\"",
                            ""))
                                .unwrap_or(value_key.to_owned());
                            name_parts.push(name)
                        }
                    }
                }
                _ => (),
            }
        }
    }

    if let Some(key) = &*name.key {
        match key.as_str() {
            "%ADJECTIVE%" | "%ADJ%" => (),
            _ => name_parts.push(key.to_string()),
        }
    }
    let mut joined: String = name_parts.join(" ");
    for prefix in NAME_PREFIX {
        joined = joined.replace(prefix,
        "");
    }

    Ok(Some(
        joined.split("_").into_iter().collect: :<Vec<_>>().join(" "),
    ))
}

Utilizando Rust tenho uma função que tem o intuito de receber um PurpleName presente em gamestate_model.rs
e converter para uma string de nome legivel com base no 99_README_GRAMMAR.txt. Entretanto ela não cobre todos
os casos de uso. 

Escreva uma função que leve em consideração a estrutura de dados de nome presente em gamestate_mode.rs
e as regras contidas no arquivo 99_README_GRAMMAR.txt, leia as regras e com base nele converta pra nomes legiveis.

As "keys" como SPEC_Polvanite ou SPEC_Lavis e semelhantes como NAME_Autonomous_Repurposer estão presentes 
em um HashMap criado a partir de species_01_l_english.yml e arquivos semelhantes chamados de arquivos de localização
A formatação consiste em ler o dado de entrada e quando for encontrado uma chave contida no hashmap ela 
deve ser substituída pelo valor referente a chave. 
Por exemplo 
"{\"key\":\"%ADJECTIVE%\",\"literal\":null,\"variables\":[{\"value\":{\"key\":\"SPEC_Nharr\",\"variables\":null},\"key\":\"adjective\"},{\"value\":{\"key\":\"Society_Name\",\"variables\":null},\"key\":\"1\"}]}". 
SPEC_Nharr e Society_name estão presentes no HashMap e seus valores são: Nharr e Society
Então o resultado da formatação do exemplo deve ser algo como: "Nharr Society"

Casos de uso estão exemplificados a seguir em formato json, porem pode-se assumir que os dados estarão 
convertidos na struct PurpleName presente no arquivo gamestate_model.rs

- Como devemos tratar os campos literal e variables se a chave não for %ADJECTIVE% ou %ADJ%?

Se a chave não for %ADJECTIVE% ou %ADJ% deve-se verificar se o campo literal contem "yes"
"{\"key\":\"Dark Angels Of The Apocalypse\",\"literal\":\"yes\",\"variables\":null}"
no exemplo a seguir o nome final formatado seria a propria Key: "Dark Angels Of The Apocalypse"

nos casos em que a chave não for %ADJECTIVE% ou %ADJ%, o campo literal é null e variables também é null
deve ser buscar a key no HashMap de localização
"{\"key\":\"NAME_Muutagan_Merchant_Guild\",\"literal\":null,\"variables\":null}"
neste caso o resultado final seria o valor presente no HashMap: "Muutagan Merchant Guild"


- O que devemos fazer com as outras variables que não são adjective ou 1?

Deve-se verificar se há value, se houver, então value.key deve ser checado no hashmap. Como no exemplo a baixo

o resultado final seria: Brea Lora Republic
